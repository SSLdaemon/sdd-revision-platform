For Software Design & Development (SDD) and Information Processes & Technology (IPT)
NESA Pseudocode Requirements
NESA requires a structured, keyword-based pseudocode style with strict formatting rules. All keywords must be in CAPS, every opening structure must have a closing pair, and code must be consistently indented.
Core Keywords & Control Structures
Procedure Definition
BEGIN ProcedureName[statements]END
Example:BEGIN CalculateAreaINPUT Length, WidthArea = Length * WidthDISPLAY AreaEND
Selection (IF-THEN-ELSE)
IF condition THEN[statements]ELSE[statements]ENDIF
Example:IF Hours > NormalHours THENOvertime = Hours - NormalHoursPay = (NormalHours * Rate) + (Overtime * Rate * 1.5)ELSEPay = Hours * RateENDIF
Multi-way Selection (CASE)
CASEWHERE variablevalue1: [statements]value2: [statements]OTHERWISE: [statements]ENDCASE
Example:CASEWHERE Grade'A': DISPLAY "Excellent"'B': DISPLAY "Good"'C': DISPLAY "Satisfactory"OTHERWISE: DISPLAY "Needs Improvement"ENDCASE
Pre-test Loop (WHILE)
WHILE condition[statements]ENDWHILE
Example:Counter = 1WHILE Counter <= 10DISPLAY CounterCounter = Counter + 1ENDWHILE
Post-test Loop (REPEAT-UNTIL)
REPEAT[statements]UNTIL condition
Example:REPEATINPUT PasswordAttempts = Attempts + 1UNTIL Password = "secret" OR Attempts >= 3
Count Loop (FOR)
FOR variable = start TO end[statements]NEXT
Example:Total = 0FOR Counter = 1 TO 50Total = Total + CounterNEXTDISPLAY Total

Complete Worked Example: Pay Calculator
Problem: Calculate employee pay with overtime (1.5× rate for hours over 7).
BEGIN CalculatePayBaseRate = 50NormalHours = 7Overtime = 0
INPUT HoursIF Hours > NormalHours THEN    BaseHours = NormalHours    Overtime = Hours - NormalHours    TotalPay = (BaseHours * BaseRate) + (Overtime * BaseRate * 1.5)ELSE    BaseHours = Hours    TotalPay = BaseHours * BaseRateENDIFDISPLAY "Total Pay: $", TotalPay
END
Essential Formatting Rules
CAPS for all keywords — IF, THEN, ELSE, ENDIF, WHILE, ENDWHILE, FOR, NEXT, BEGIN, END, etc.
Matching pairs — Every opening structure must have a closing keyword:
IF  ENDIF
WHILE  ENDWHILE
FOR  NEXT
BEGIN  END
CASEWHERE  ENDCASE
REPEAT  UNTIL
Consistent indentation — Indent all statements inside control structures by 4 spaces or 1 tab
Assignment uses = — Standard equals sign for variable assignment (e.g., Total = 0)
Boolean operators in CAPS — AND, OR, NOT
Initialize variables — Always set initial values before use
Clear variable names — Use descriptive names (e.g., TotalPay not TP)
Common Mistakes to Avoid
Missing ENDIF/ENDWHILE/NEXT — Every opening keyword must close
Lowercase keywords — Write IF not if
Inconsistent indentation — Keep structure visible
Using = for comparison — Use correct comparison operators (>, <, >=, <=, =)
Forgetting to initialize counters — Set Counter = 0 before loops
Not closing files — Always CLOSE after file operations
Teaching Progression
Stage 1: Basic Sequences
Teach BEGIN/END structure
Practice INPUT, DISPLAY, and simple calculations
Introduce variable assignment and initialization
Stage 2: Selection
Start with simple IF-THEN-ENDIF (no ELSE)
Progress to IF-THEN-ELSE-ENDIF
Introduce CASEWHERE for multiple options
Stage 3: Iteration
Begin with FOR loops (fixed count)
Move to WHILE loops (pre-test)
Teach REPEAT-UNTIL (post-test)
Stage 4: File Processing
Introduce OPEN, READ, WRITE, CLOSE
Practice EOF checking with WHILE NOT EOF
Combine file operations with selection and iteration
Assessment Expectations
Students must demonstrate:
Correct use of CAPS keywords
Properly paired opening/closing structures
Consistent, clear indentation
Logical flow from problem to solution
Appropriate use of selection and iteration
Correct initialization of variables
Meaningful variable names
