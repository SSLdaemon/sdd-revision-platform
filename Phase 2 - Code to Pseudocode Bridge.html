Before we begin
What is Pseudocode?
Pseudocode describes program logic using structured keywords instead of a specific programming language. In NSW, we follow the NESA pseudocode standard.
🔠

CAPS Keywords

Keywords like IF, DISPLAY, INPUT are always written in capitals

🧩

Matching Pairs

Every IF needs an ENDIF, every BEGIN needs an END

🎯

Logic First

Focuses on what the program does, not how it’s written in any particular language

The challenge
Why Reverse-Engineer?
Most people learn to write pseudocode first, then code. Today we’re doing it backwards — and that’s on purpose.
🔄

When you read working code and have to extract the logic yourself, you build a much deeper understanding of how programs work. You’re not just following instructions — you’re figuring out the thinking behind the code.

This is the same skill professional developers use when they need to understand someone else’s code — which happens all the time in the real ICT industry.

Things to look for
Reading Code: What to Notice
🔍 Pattern 1: DISPLAY

Python’s print("...") maps to the NESA keyword DISPLAY. Multiple prints in a row may be one DISPLAY step.

🔍 Pattern 2: INPUT

Python’s input("...") maps to INPUT. The result is stored in a variable — name it clearly.

🔍 Pattern 3: IF…THEN…ENDIF

Python’s if / elif / else maps to IF...THEN / ELSE IF...THEN / ELSE / ENDIF. Every IF needs an ENDIF.

🔍 Pattern 4: ELSE

ELSE catches everything that doesn’t match. Don’t forget to close the whole block with ENDIF.
